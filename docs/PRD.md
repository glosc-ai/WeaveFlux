# 织影 (WeaveFlux)

## 产品需求文档 (PRD)

### 1. 核心业务流程

由于应用无任何远程服务器支持，整个业务流在本地由 Flutter UI 与本地 Go 核心层协作完成：

1. **初始配置：** 用户首次打开 APP 需进入“设置”页面，手动填入符合 OpenAI 规范的 `BaseURL` 和 `API Key`，否则创作功能处于锁定状态。
2. **任务发起：** 用户输入文字 Prompt 或上传本地图片，调整对应参数（如分辨率、比例、模型名称），点击“开始生成”。
3. **本地发送：** APP 直接在本地校验参数完整性，由嵌入式 Go 核心层封装符合 OpenAI 规范的 HTTP 请求投递给 `glosc ai one` 接口。
4. **异步状态轮询：** Go 核心层捕获接口返回的 Task ID，在本地开启定时器（Ticker）进行异步轮询。
5. **结果持久化：** 轮询成功后获取到视频的远端 URL，APP 自动将其下载到 Android 本地沙盒，更新本地嵌入式数据库的状态，并通知用户可以保存至系统相册。

### 3. 非功能性需求与安全规范

* **密钥存储安全性：** API Key 严禁以明文形式存入普通的 SharedPreferences。必须调用 Flutter Secure Storage，在 Android 端底层的实现是基于 **Android Keystore System** 进行 AES-256 加密。
* **存储空间管理：** APP 应在画廊提供“清理缓存”功能，删除沙盒内视频以释放空间，但已导出到系统相册的视频不受影响。
* **网络鲁棒性：** 后台 Go 轮询模块必须具备指数退避（Exponential Backoff）的重试策略，在网络临时中断时自动挂起，恢复后继续轮询。
